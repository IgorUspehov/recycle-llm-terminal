# LLM Backends
Ollama
local · self-hosted · no cloud
Run quantized LLMs locally on CPU/GPU. No API key, no rate limits. Primary backend for all polymer engineering prompts.
$ ollama run mistral "prompt"
$ ollama run codellama "python script..."
$ ollama list
llm CLI
Simon Willison · pip install llm
Unified CLI wrapper for multiple backends (OpenAI, Claude, Ollama plugin). Enables piping and shell integration: cat data.txt | llm "analyse".
$ llm "calculate MFI drift..."
$ cat batch.csv | llm "find anomalies"
$ llm logs --since 1d
LocalAI
OpenAI-compatible · REST API
Drop-in OpenAI API replacement running locally. Used when scripts need requests.post("/v1/chat/completions") calls.
$ localai run models/mistral.gguf
$ curl localhost:8080/v1/chat/completions
# Models Used
Mistral 7B
GGUF Q4_K_M · ~4.1 GB RAM
Main workhorse for polymer formulation, temperature calculation, stabilizer selection. Excellent instruction-following, fast on CPU.
$ ollama run mistral
CodeLlama 7B
code-instruct · Python focus
Used to generate Python scripts: extruder monitoring, CSV parsing, amperage deviation alerts, MFI trend plots.
$ ollama run codellama
DeepSeek Coder 6.7B
code · instruct · fast
Secondary code model. Better than CodeLlama on structured data parsing tasks. Used for bash scripts and ADB automation where needed.
$ ollama run deepseek-coder
Claude API (claude.ai)
cloud · via browser / API
Used for longer, complex formulation reasoning, multi-step diagnostics, or when local RAM is the bottleneck.
$ llm -m claude-3-5-sonnet "..."
